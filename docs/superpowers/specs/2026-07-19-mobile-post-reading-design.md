# 모바일 글 읽기 화면 개선 설계

작성일: 2026-07-19

## 배경

사이트의 목표는 포트폴리오와 기술 블로그를 균형 있게 운영하는 것이다. 현재 글 상세 화면은 데스크톱에서 본문과 사이드바를 두 열로 배치하지만, 모바일에서는 두 요소가 같은 그리드 셀에 놓여 서로 겹친다. 글을 읽을 수 없게 만드는 문제이므로 다른 포트폴리오 개선보다 먼저 해결한다.

## 확인된 근본 원인

`assets/css/style.css`의 데스크톱 규칙은 다음 배치를 사용한다.

- `.post-content`: 첫 번째 열, 첫 번째 행
- `.post-sidebar`: 두 번째 열, 첫 번째 행

화면 폭이 720px 이하가 되면 `.post-layout`은 한 열로 변경되고 `.post-sidebar`의 열만 초기화된다. 그러나 본문과 사이드바의 `grid-row: 1`은 그대로 남으므로 두 요소가 한 열의 첫 번째 행에서 겹친다.

이 회귀는 `f9be416` 커밋에서 두 요소에 `grid-row: 1`을 추가하면서 발생했다.

## 목표

- 375px 이상의 일반적인 모바일 화면에서 글 본문이 다른 UI와 겹치지 않는다.
- 모바일 사용자가 제목에서 본문까지 방해 없이 읽고, 필요할 때 목차를 펼칠 수 있다.
- 코드, 긴 기술 용어, 외부 링크가 화면 전체 폭을 늘리지 않는다.
- 데스크톱의 두 열 읽기 구조는 유지한다.
- 다크 모드와 키보드 탐색에서도 본문과 조작 요소를 명확히 인식할 수 있다.
- 새로운 프레임워크나 외부 JavaScript 의존성을 추가하지 않는다.

## 범위

### 포함

- 글 상세 템플릿 `_layouts/post.html`
- 공통 레이아웃의 목차 생성 및 테마 처리 스크립트 `_layouts/default.html`
- 글 상세 화면과 모바일 헤더에 필요한 `assets/css/style.css`
- 모바일 겹침 회귀 검증
- 대표 글을 이용한 반응형·접근성 확인

### 제외

- 홈과 프로젝트 페이지의 전면 재설계
- 글 내용과 태그 체계 개편
- Jekyll 또는 다른 정적 사이트 생성기로의 마이그레이션
- 히어로 이미지 최적화와 SEO 확장
- 데스크톱 디자인의 시각적 전면 변경

## 정보 구조

글 상세 화면의 문서 순서는 다음과 같이 정리한다.

1. 글 제목, 날짜, 작성자 역할, 요약, 태그
2. 글 목차
3. 글 본문
4. 주제·탐색 보조 링크
5. 관련 글
6. 이전 글과 다음 글

현재 하나의 사이드바에 들어 있는 목차와 보조 링크를 별도 요소로 분리한다. 이렇게 하면 CSS의 강제 행 배치에 의존하지 않고 모바일에서 자연스러운 문서 순서를 만들 수 있다.

## 반응형 레이아웃

### 데스크톱: 721px 이상

- 본문은 왼쪽 열, 목차와 보조 링크는 오른쪽 열에 배치한다.
- 목차는 현재처럼 화면을 따라오는 형태를 유지한다.
- 본문 최대 폭과 전체 1120px 컨테이너는 유지한다.
- 관련 글과 이전·다음 글의 기존 다열 구조를 유지한다.

### 모바일: 720px 이하

- `.post-layout`을 명시적인 한 열 흐름으로 바꾼다.
- 모든 글 상세 그리드 항목의 `grid-column`과 `grid-row`를 모바일 규칙에서 초기화한다.
- 순서는 목차, 본문, 보조 링크로 고정한다.
- 목차는 `<details open>`과 `<summary>`로 구성한다. 데스크톱에서는 기본 펼침 상태를 유지하고, 기존 스크립트가 모바일 최초 로드에서만 `open`을 제거해 기본 접힘 상태로 만든다.
- 상단 헤더는 세로로 두 줄 쌓지 않고 약 56px 높이의 한 줄 구조로 축소한다.
- 모바일 헤더에는 로고, Posts, Projects, 테마 버튼을 남긴다. 로고의 보조 문구와 GitHub 링크는 헤더에서 숨기되 GitHub 링크는 푸터에서 계속 제공한다.
- 본문 좌우 여백은 20px을 기준으로 유지한다.
- 관련 글과 이전·다음 글은 한 열 카드로 표시한다.

## 읽기 경험

- 모바일 본문 기본 크기는 16px, 줄 간격은 1.75를 사용한다.
- 모바일 글 제목은 30px로 조정해 긴 제목이 화면 대부분을 차지하지 않게 한다.
- 모바일 `h2`는 24px, `h3`는 20px로 조정하고 위쪽 간격을 줄이되 섹션 구분은 유지한다.
- 본문 문단, 목록, 인라인 코드에는 `overflow-wrap` 규칙을 적용해 긴 기술 용어가 뷰포트를 넓히지 않게 한다.
- 코드 블록은 자체 가로 스크롤을 유지하고 모바일 패딩과 글자 크기를 줄인다.
- 태그와 조작 버튼의 터치 영역은 최소 44px을 목표로 한다.

## 접근성 및 테마

- 모든 링크, 버튼, `<summary>`에 명확한 `:focus-visible` 표시를 제공한다.
- 모바일 헤더의 테마 버튼도 최소 44px 터치 영역을 확보한다.
- 테마 버튼에 `aria-pressed`를 적용하고 현재 상태에 맞게 접근 가능한 이름을 갱신한다.
- 다크 모드에서 고정된 `#555555` 본문 색상을 의미 기반 색상 변수로 교체해 WCAG AA 대비를 충족한다.
- 목차가 비어 있으면 현재 동작처럼 숨긴다.
- 애니메이션을 추가하지 않으며, 향후 추가될 경우 `prefers-reduced-motion`을 따른다.

## 동작과 예외 처리

- 기존 목차 생성 코드는 본문의 `h2`를 기준으로 링크를 만든다.
- 제목에 ID가 없으면 순서에 따라 안정적인 ID를 부여한다.
- `h2`가 없는 글은 목차 전체를 숨기고 빈 조작 요소를 남기지 않는다.
- JavaScript가 동작하지 않아도 본문 자체는 정상 순서로 노출되어야 한다.
- 모바일과 데스크톱 사이에서 화면 크기가 바뀌어도 요소가 겹치지 않아야 한다.

## 구현 단위

1. 모바일 겹침을 재현하는 회귀 검사를 만든다.
2. 목차와 보조 링크의 템플릿 구조를 분리한다.
3. 데스크톱과 모바일 그리드 배치를 명시한다.
4. 모바일 헤더와 본문 타이포그래피를 조정한다.
5. 코드·긴 문자열의 오버플로를 방지한다.
6. 다크 모드 대비, 포커스, 터치 영역을 보완한다.
7. 대표 글과 전체 사이트 기본 검증을 실행한다.

## 검증 기준

다음 화면 폭을 기준으로 확인한다.

- 375px
- 390px
- 430px
- 720px
- 721px
- 768px
- 1024px 이상

각 화면에서 다음 조건을 만족해야 한다.

- 본문, 목차, 보조 링크가 서로 겹치지 않는다.
- 문서 전체의 가로 스크롤이 생기지 않는다.
- 코드 블록만 필요한 경우 자체적으로 가로 스크롤한다.
- 목차를 키보드와 터치로 열고 닫을 수 있다.
- 글 제목과 첫 문단이 고정 헤더에 가려지지 않는다.
- 라이트 모드와 다크 모드에서 본문을 읽을 수 있다.
- 관련 글과 이전·다음 글 링크의 터치 영역이 충분하다.
- 768px 이상의 데스크톱 구조는 기존 두 열 배치를 유지한다.

대표 검증 글은 다음 유형을 포함한다.

- 긴 제목이 있는 최신 글
- 코드 블록이 여러 개 있는 글
- 긴 외부 링크와 인라인 코드가 있는 글
- 관련 글과 troubleshooting 경로가 모두 표시되는 글

## 완료 조건

- 모바일 겹침을 재현하는 검사가 수정 전 실패하고 수정 후 통과한다.
- 정적 검사와 가능한 Jekyll 빌드가 통과한다.
- 배포 전 소스에서 모바일·데스크톱 레이아웃 규칙이 모두 확인된다.
- 배포 후 대표 글과 사이트맵 URL이 정상 응답한다.
- 이번 변경에 홈·프로젝트 전면 개편이나 새로운 의존성이 섞이지 않는다.
