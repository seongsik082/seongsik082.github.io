Java 문법은 처음에는 단순해 보이지만, 객체지향 코드와 Spring 애플리케이션을 작성하다 보면 기본 개념의 차이가 중요해집니다.
이 글은 Java를 다시 정리할 때 먼저 확인하면 좋은 문법 요소를 모아둔 노트입니다.

## 값 타입과 참조 타입

Java의 타입은 크게 기본형과 참조형으로 나눌 수 있습니다.

기본형은 실제 값을 변수에 저장합니다.

```java
int age = 20;
boolean active = true;
double score = 95.5;
```

참조형은 객체가 있는 위치를 참조합니다.

```java
User user = new User("seongsik");
String name = "kim";
```

이 차이는 메서드에 값을 넘길 때 특히 중요합니다. 기본형은 값이 복사되고, 참조형은 참조값이 복사됩니다.
그래서 객체 내부 상태를 변경하면 호출한 쪽에서도 변경된 상태를 볼 수 있습니다.

## 접근 제어자

접근 제어자는 클래스와 멤버를 어디까지 공개할지 결정합니다.

- `private`: 같은 클래스 안에서만 접근
- `default`: 같은 패키지 안에서 접근
- `protected`: 같은 패키지 또는 상속 관계에서 접근
- `public`: 어디서나 접근

백엔드 코드에서는 무조건 `public`으로 열기보다 필요한 범위만 공개하는 습관이 좋습니다.
도메인 객체의 상태 변경 메서드를 제한하면 잘못된 사용을 줄일 수 있습니다.

## static

`static`은 인스턴스가 아니라 클래스에 속한다는 뜻입니다.

```java
public class UserValidator {
    public static boolean isValidName(String name) {
        return name != null && !name.isBlank();
    }
}
```

상태를 가지지 않는 유틸성 메서드에는 유용하지만, 애플리케이션 전역 상태를 `static`으로 들고 있으면 테스트와 동시성에서 문제가 생길 수 있습니다.

## final

`final`은 재할당, 재정의, 상속을 막을 때 사용합니다.

```java
final int maxRetry = 3;
```

변수에 붙이면 재할당을 막고, 메서드에 붙이면 오버라이딩을 막고, 클래스에 붙이면 상속을 막습니다.
의도를 코드에 남기는 장치라고 생각하면 이해하기 쉽습니다.

## 정리

Java 문법을 잘 안다는 것은 키워드를 외우는 것이 아니라 코드의 변경 가능성과 공개 범위를 조절할 수 있다는 뜻입니다.
Spring이나 JPA를 공부할 때도 이 기본기가 있으면 프레임워크의 동작을 더 잘 이해할 수 있습니다.
