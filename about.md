  <h2>소개</h2>
  <p>
    안녕하세요. 백엔드 개발자 김성식입니다. 서비스의 요청 흐름, 데이터 모델, 장애 가능성,
    운영 자동화에 관심이 있습니다. 기능을 빠르게 만드는 것뿐 아니라 나중에 읽히고 고칠 수 있는
    구조를 만드는 개발자가 되고 싶습니다.
  </p>
  <p>
    이 블로그에서는 "어떤 개념인가"보다 "어디서 문제를 만나고, 어떤 기준으로 선택하며,
    운영에서 무엇을 먼저 확인해야 하는가"를 함께 남기려고 합니다.
  </p>

  <h2>일하는 방식</h2>
  <ul>
    <li>요구사항을 API, 데이터, 예외 흐름으로 나누어 정리합니다.</li>
    <li>동작하는 코드 이후에 테스트, 로그, 배포 흐름까지 확인합니다.</li>
    <li>트러블슈팅 과정은 재현 조건, 원인, 해결, 재발 방지 순서로 기록합니다.</li>
  </ul>

  <h2>블로그에서 자주 다루는 영역</h2>
  <p>
    Java, Spring Boot, JPA, HTTP API, 데이터베이스, 캐시, 비동기 처리, 배포 자동화,
    관측성과 성능 이슈를 중심으로 글을 쌓고 있습니다.
  </p>

  <h2>이 블로그를 어떻게 읽으면 좋은가</h2>
  <p>
    처음 방문했다면 Posts에서 Java, Spring, Database, REST API 태그부터 보는 것을 추천합니다.
    글마다 정의만 정리하지 않고, 실수 패턴과 운영에서 확인할 지표를 같이 남기려 합니다.
  </p>
