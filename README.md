# GitHub Blog Starter

백엔드 개발자 포트폴리오 겸 기술 블로그를 위한 GitHub Pages starter입니다.

## 사용 방법

1. GitHub에서 `seongsik082.github.io` 이름의 public repository를 만듭니다.
2. 이 폴더의 파일을 repository 루트에 올립니다.
3. `_config.yml`에서 이름, 이메일, GitHub ID를 수정합니다.
4. `_posts` 폴더에 `YYYY-MM-DD-title.md` 형식으로 글을 추가합니다.
5. GitHub repository의 `Settings > Pages`에서 배포 상태를 확인합니다.

## 추천 글 주제

- JWT 인증/인가 흐름 정리
- JPA N+1 문제를 발견하고 해결한 과정
- MySQL 인덱스 적용 전후 비교
- Redis 캐싱을 적용할 때 고려한 만료 전략
- Docker와 GitHub Actions로 배포 자동화하기
- 장애나 버그를 재현하고 고친 과정

## 수정하면 좋은 곳

- `index.md`: 첫 화면 소개 문구
- `about.md`: 자기소개와 기술 스택
- `projects.md`: 대표 프로젝트
- `_config.yml`: 사이트 이름, 이메일, GitHub ID
- `assets/css/style.css`: 색상과 레이아웃
